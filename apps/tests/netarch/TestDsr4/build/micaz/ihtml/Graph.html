Interface: Graph

interface Graph

Basic operations for processsing and modifying a graph.

Author:
Ryan Stinnett

Commands - Details

directed

command bool directed()

Checks if the graph is directed or not. This value should not change at run-time.

Returns:
TRUE if the graph is directed, FALSE if it is undirected.

edge

command graph_edge_id_t edge(graph_vertex_id_t s, graph_vertex_id_t d)

Retrieves an edge between vertices s and d, if it exists. If the graph is directed, s is taken to be the source and d is taken to be the destination of the directed edge.

Parameters:
s - The first vertex or source of the edge.
d - The second vertex or destination of the edge.
Returns:
The edge key if the edge was found, maxEdges() if it was not.

edgeList

command graph_edge_id_t edgeList(graph_vertex_id_t s)

Retrieves a list of edges adjacent to vertex s. If the graph is directed, s is taken to be the source of the directed edge.

Parameters:
s - The first vertex or source of the edge.
Returns:
The key to the first node of the list if the vertex was valid, maxEdges() if it was not.

edges

command graph_edge_id_t edges()

Gets the number of edges currently in the graph. This should return the number of "conceptual" edges. For example, an undirected graph might use two directed edges for each undirected edge in its implementation, but this only counts as one edge here. For some implementations, this will always equal maxEdges() if all edges are preallocated.

Returns:
The number of edges currently in the graph.

insert

command graph_edge_id_t insert(graph_vertex_id_t s, graph_vertex_id_t d)

Add a new edge to the graph between vertices s and d.

Parameters:
s - The first vertex or source of the edge.
d - The second vertex or destination of the edge.
Returns:
If all of the edges have already been allocated or if parallel edges are not supported and this edge has already been added or if either vertex is too large, then maxEdges() is returned. Otherwise, the key for the new edge is returned.

maxEdges

command graph_edge_id_t maxEdges()

Gets the maximum number of edges that the graph can hold. This should return the number of "conceptual" edges. For example, an undirected graph might use two directed edges for each undirected edge in its implementation, but this only counts as one edge here.

Returns:
The maximum number of edges that the graph can hold.

maxVertices

command graph_vertex_id_t maxVertices()

Gets the maximum number of vertices that the graph can hold.

Returns:
The maximum number of vertices that the graph can hold.

parallelEdges

command bool parallelEdges()

Checks if parallel edges are allowed or not. This value should not change at run-time.

Returns:
TRUE if parallel edges are allowed, FALSE if they are not.

remove

command error_t remove(graph_vertex_id_t s, graph_vertex_id_t d)

Remove an edge from the graph between vertices s and d.

Parameters:
s - The first vertex or source of the edge.
d - The second vertex or destination of the edge.
Returns:
SUCCESS if the edge was found and removed, EINVAL if it was not. ESIZE returned if either vertex is too large.

vertices

command graph_vertex_id_t vertices()

Gets the number of vertices currently in the graph. For some implementations, this will always equal maxVertices() if all vertices are preallocated.

Returns:
The number of vertices currently in the graph.

Events - Details

inserted

event void inserted(graph_edge_id_t m)

The structure of the graph changed because an edge was inserted.

Parameters:
m - The key for the edge that was inserted.

removed

event void removed(graph_edge_id_t m)

The structure of the graph changed because an edge was removed.

Parameters:
m - The key for the edge that was removed.
