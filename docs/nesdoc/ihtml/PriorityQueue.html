Interface: PriorityQueue

interface PriorityQueue

Interface to a priority queue sitting on top of an index heap. The meaning of some terms defined here is implementation dependent since either a min or max heap could be used.

Author:
Ryan Stinnett

Commands
command void clear() Removes all elements from the priority queue.
command bool empty() Checks if the priority queue is empty.
command priority_queue_id_t findBest() Finds the index of the element with the best priority (at the root of the heap), but does not remove it.
command error_t insert(priority_queue_id_t m) Insert the element at index m into the priority queue.
command bool inserted(priority_queue_id_t m) Checks whether element m has already been inserted.
command priority_queue_id_t maxSize() The maximum number of elements the priority queue can hold.
command void move(priority_queue_id_t m) If the priority of element m has changed, move that element up or down the heap if needed.
command void moveDown(priority_queue_id_t m) If the priority of element m has worsened, move that element down the heap if needed.
command void moveUp(priority_queue_id_t m) If the priority of element m has improved, move that element up the heap if needed.
command priority_queue_id_t removeBest() Finds the index of the element with the best priority (at the root of the heap) and removes it from the priority queue.
command priority_queue_id_t size() The number of elements currently in the priority queue.

Commands - Details

clear

command void clear()

Removes all elements from the priority queue.

empty

command bool empty()

Checks if the priority queue is empty.

Returns:
Whether the priority queue is empty.

findBest

command priority_queue_id_t findBest()

Finds the index of the element with the best priority (at the root of the heap), but does not remove it.

Returns:
Index of the element with the best priority.

insert

command error_t insert(priority_queue_id_t m)

Insert the element at index m into the priority queue.

Parameters:
m - Index of the element to insert.
Returns:
If the priority queue is full then FAIL, if m is too large for the queue then ESIZE, if m has already been inserted then EALREADY, otherwise SUCCESS.

inserted

command bool inserted(priority_queue_id_t m)

Checks whether element m has already been inserted.

Parameters:
m - Index of the element to check on.
Returns:
TRUE if element m has already been inserted, otherwise FALSE.

maxSize

command priority_queue_id_t maxSize()

The maximum number of elements the priority queue can hold.

Returns:
The maximum priority queue size.

move

command void move(priority_queue_id_t m)

If the priority of element m has changed, move that element up or down the heap if needed.

Parameters:
m - Index of the element with altered priority.

moveDown

command void moveDown(priority_queue_id_t m)

If the priority of element m has worsened, move that element down the heap if needed.

Parameters:
m - Index of the element with altered priority.

moveUp

command void moveUp(priority_queue_id_t m)

If the priority of element m has improved, move that element up the heap if needed.

Parameters:
m - Index of the element with altered priority.

removeBest

command priority_queue_id_t removeBest()

Finds the index of the element with the best priority (at the root of the heap) and removes it from the priority queue.

Returns:
Index of the element with the best priority.

size

command priority_queue_id_t size()

The number of elements currently in the priority queue. Always less than or equal to maxSize().

Returns:
The number of elements in the priority queue.
