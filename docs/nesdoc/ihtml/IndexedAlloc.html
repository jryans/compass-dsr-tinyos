Interface: IndexedAlloc

interface IndexedAlloc

Interface to a component that implements a run-time allocation policy on top of a static size array via indices. This simplifies component design by moving the policy out of each component, leading to cleaner code.

Author:
Ryan Stinnett

Commands
command bool all() Checks if all of the elements are allocated.
command indexed_alloc_id_t alloc() Allocate an index in the set.
command bool allocated(indexed_alloc_id_t m) Checks if an index is currently allocated.

Commands - Details

all

command bool all()

Checks if all of the elements are allocated.

Returns:
TRUE if all of the elements are allocated, FALSE otherwise.

alloc

command indexed_alloc_id_t alloc()

Allocate an index in the set.

Returns:
If an index is free, it will be allocated and returned. Otherwise, maxSize() is returned to indicate that all elements are allocated.

allocated

command bool allocated(indexed_alloc_id_t m)

Checks if an index is currently allocated.

Returns:
TRUE if the index is allocated, FALSE if it is not.

free

command error_t free(indexed_alloc_id_t m)

Frees an index in the set.

Returns:
If an index is allocated, it will be freed and SUCCESS is returned. If it was already free, no change is made and EALREADY is returned. If the index is too large, ESIZE is returned.

maxSize

command indexed_alloc_id_t maxSize()

The maximum number of elements that can be allocated.

Returns:
The maximum size that can be allocated.

none

command bool none()

Checks if none of the elements are allocated.

Returns:
TRUE if none of the elements are allocated, FALSE otherwise.

size

command indexed_alloc_id_t size()

The number of elements currently allocated. Always less than or equal to maxSize().

Returns:
The number of elements currently allocated.
